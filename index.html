Help & Reference

Prior P(C)
How common each class is in training data. Computed before seeing any features.
Likelihood P(X|C)
How probable the observed features are, given a specific class.
Posterior P(C|X)
Final class probability after combining prior and likelihood using Bayes theorem.
Naive Assumption
All features are treated as conditionally independent given the class label.
Laplace Smoothing
Adding 1 to all counts to prevent zero-probability for unseen feature values.
Log-space
Using log(prob) to avoid numerical underflow when multiplying many small probabilities.
Gaussian PDF
Bell-curve formula: P(x|C) = (1/√2πσ²)·exp(-(x-μ)²/2σ²)
Zero-Frequency Problem
When a feature value never appeared in a class during training — makes P=0 and kills the whole product.
NB needs tabular, labeled data. CSV is the universal ML data format. Each row = one sample, columns = features, one column = the class label.
NB breaks completely with missing values (can't compute P(NaN|Class)) and with text features (can't compute Gaussian PDF of a string). Encoding and cleaning are non-negotiable.
EDA reveals class imbalance (biases prior), feature distributions (helps pick Gaussian vs Multinomial), and correlations (checks the independence assumption).
Priors tell NB the base rate of each class. Likelihoods capture how each feature value shifts the probability toward or away from each class. Together they give the posterior.
To test if the model generalises. Training accuracy is always higher — the split forces honest evaluation on data the model has never seen.
Accuracy lies on imbalanced datasets. A model that always predicts "No" on a 95% No dataset gets 95% accuracy but 0% recall. F1 catches this.
Not handling nulls — NB returns NaN for every prediction if any feature has missing values.
Text features without encoding — Gaussian PDF of a string is undefined. Always encode before training.
Gaussian NB on count data — Use Multinomial NB for word counts, frequencies, integer features.
Skipping log-space — Multiplying 50+ probabilities of 0.01 causes floating point underflow to 0.0.
Ignoring class imbalance — The prior dominates; a rare class almost never gets predicted. Balance your dataset or adjust priors.
Evaluating on training data — The model has "seen" this data. Always evaluate on the held-out test set.
Assuming independence holds — Check the correlation heatmap. Highly correlated features hurt NB's calibration.
Bayes: P(C|X) = P(X|C) × P(C) / P(X) Prior: P(C) = count(C) / N Gaussian: P(x|C) = (1/√2πσ²)·exp(-(x-μ)²/2σ²) Laplace: P(xᵢ=v|C) = (count+1) / (|C|+k) Log-score: logP(C|X) ∝ logP(C) + Σ logP(xᵢ|C) Softmax: P(C) = exp(score) / Σ exp(scores) Accuracy: (TP+TN)/(TP+TN+FP+FN) Precision: TP/(TP+FP) Recall: TP/(TP+FN) F1: 2·P·R/(P+R)
